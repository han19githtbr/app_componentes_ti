CONCEITO FUNDAMENTAL

O que é Memória Cache?

A memória cache é uma memória de acesso ultra-rápido posicionada entre o processador (CPU) e a memória RAM. Seu objetivo é reduzir a latência de acesso a dados frequentemente usados, evitando que a CPU fique parada esperando a RAM — que é muito mais lenta. Sem cache, um processador moderno ficaria ocioso até 80% do tempo aguardando dados.

Registradores~0.3 ns · bytes
L1 Cache~1 ns · 32–128 KB
L2 Cache~4 ns · 256 KB–4 MB
L3 Cache~10–30 ns · 8–64 MB
RAM (DRAM)~60–100 ns · GBs
SSD / NVMe~100 µs · TBs
// níveis de cache
L1
Cache L1
Nível 1 — O mais rápido
Integrado diretamente no núcleo do processador. Dividido em L1-I (instruções) e L1-D (dados). Latência de 4 ciclos de clock. Cada núcleo tem o seu próprio L1.
Tamanho32–128 KB
Latência~1 ns
Velocidade~3–4 TB/s
LocalizaçãoNo núcleo
L2
Cache L2
Nível 2 — Buffer intermediário
Também por núcleo na maioria das CPUs modernas. Serve como buffer entre L1 e L3. Armazena dados que não couberam no L1. Latência entre 4 e 12 ciclos.
Tamanho256 KB–4 MB
Latência~4 ns
Velocidade~1 TB/s
LocalizaçãoNo chip
L3
Cache L3
Nível 3 — Compartilhado
Compartilhado entre todos os núcleos da CPU. Permite que núcleos troquem dados sem ir à RAM. Encontrado em todos os processadores de desktop modernos. Chamado de LLC (Last Level Cache).
Tamanho8–64 MB
Latência~10–30 ns
Velocidade~300 GB/s
LocalizaçãoNo die
L4
Cache L4 / eDRAM
Nível 4 — Opcional
Presente em CPUs Intel com Iris Pro e alguns Xeon. Usa DRAM embarcado (eDRAM) como cache entre L3 e RAM. Também utilizado em alguns processadores ARM de alto desempenho.
Tamanho64–256 MB
Latência~30–50 ns
Velocidade~100 GB/s
LocalizaçãoFora do die

Políticas de Cache
Como a cache decide o que guardar e descartar
🔄 Write-Through
Toda escrita vai simultaneamente para cache e RAM. Garante consistência, mas é mais lento nas escritas. Usado em caches L1 de algumas arquiteturas.
Mais seguro
⚡ Write-Back
Dados são escritos primeiro na cache e copiados para RAM somente quando a linha é descartada. Muito mais rápido — usado em L1/L2 modernas.
Mais rápido
🗑️ LRU (Least Recently Used)
O dado menos recentemente usado é o primeiro a ser substituído quando a cache está cheia. O algoritmo de eviction mais comum em caches L1/L2/L3.
Mais comum
🎯 Prefetching
A CPU prevê quais dados serão necessários e os carrega na cache antes do pedido explícito. Reduz cache misses em padrões de acesso sequencial.
Preditivo
🔗 Associatividade
Define quantos locais na cache um dado pode ocupar. Caches direct-mapped são simples mas com mais conflito; N-way set-associative é o padrão moderno.
Arquitetural
🧹 Cache Flush
Operação que força o esvaziamento da cache, escrevendo os dados sujos de volta à RAM e invalidando todas as linhas. Ocorre em context switch, mudança de processo crítico e reinicialização.
